## User
基于已有的rtl代码，写一个tcl脚本。 
 结论：**必须先做静态验证（语法/编译/规范/约束预检），不能直接仿真**；静态无错后，再进入动态仿真。 
 
 ### 一、先做：静态验证（不跑波形，快速排错，必做） 
 目的：把**语法、连接、规范、约束低级错**挡在仿真前，避免仿真跑不起来、波形乱、白浪费时间。 
 
 #### 1. 语法+编译检查（Vivado一键做） 
 - 操作：Vivado → 选中所有RTL → **Synthesize（综合）**，只跑综合、不跑实现。 
 - 查什么： 
   - 语法错：拼写、分号、关键字错误； 
   - 模块/端口错：例化缺失、端口位宽不匹配、未定义信号； 
   - 初步latch：组合逻辑没写`else/default`，综合会报**Latch inferred**； 
   - 未连接端口：悬空信号、未例化模块。 
 
 #### 2. 设计规范检查（贴合你的项目要求） 
 对照项目规则，快速扫一遍： 
 - 所有`if/case`都有`else/default`（**绝对不能有latch**）； 
 - 脉冲模块：`pulse_0~3`最小宽度≥2ns； 
 - DA部分：`da_clk/da_data`已设`DONT_TOUCH`； 
 - AXI-Lite：寄存器位宽、地址对齐（32位、0x43C00000区间）。 
 
 #### 3. 约束预检（导入XDC，查冲突） 
 - 导入`timing_constraints.xdc`+`pin_assignments.xdc`； 
 - 查：引脚冲突、Bank电压域跨接、时钟约束名/引脚匹配、IO标准（LVCMOS33）一致。 
 
 ### 二、做完静态，再进入：动态仿真验证（跑波形，下一步） 
 静态无报错后，再做仿真，此时仿真专注**功能与时序逻辑**，效率最高： 
 1. 写Testbench（或用已有的`*_tb.v`）； 
 2. 运行`run_all_sim.tcl`自动化仿真； 
 3. 看波形：时钟生成、脉冲宽度/延迟、AD采集、AXI读写是否正常。 
 
 ### 一句话总结 
 **静态验证（编译+规范+约束预检）→ 无错 → 动态仿真**。  
 直接跳仿真=把低级错误带进仿真，返工更慢。
